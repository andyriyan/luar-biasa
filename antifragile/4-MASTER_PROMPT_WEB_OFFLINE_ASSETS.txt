## KONTEKS WAJIB — ARSITEKTUR TERBARU

Aplikasi ini adalah PWA mobile-first, Android-native feel, immersive learning.
Bukan blog, bukan PDF reader. Materi harus one focus at a time, progressive disclosure.

### Halaman Utama Aplikasi:
| Halaman | File | Fungsi |
|---|---|---|
| Beranda | `index.html` | Hero branding madrasah + grid materi |
| Daftar Materi | `materi.html` | List materi + search + filter |
| Progress | `progress.html` | Tracking progress belajar siswa |
| Pengaturan | `settings.html` | Tema, identitas madrasah |
| Kuis | `quiz.html` | Kuis interaktif (dipanggil dari materi) |

### Bottom Navigation (4 item):
```
Beranda → index.html
Materi  → materi.html   ← BUKAN folder /materi/, tapi file materi.html
Progress → progress.html
Pengaturan → settings.html
```

## ATURAN SUPER PENTING UNTUK GAMBAR & ASSET

1. DILARANG mengarang URL gambar palsu.
2. DILARANG memaksa hosting eksternal sebagai satu-satunya cara.
3. Prioritas utama: **path lokal** di folder materi.
4. Jika gambar belum tersedia, WAJIB taruh placeholder:
   `[[LETAKKAN_GAMBAR: nama-file.webp | keyword pencarian | deskripsi fungsi gambar]]`

Contoh:
`[[LETAKKAN_GAMBAR: diagram-tekanan-hidrostatis.webp | "diagram tekanan hidrostatis SMP" | visual proses tekanan pada zat cair]]`

5. Selain placeholder di konten, WAJIB buat daftar tugas aset (lihat FORMAT BALASAN).

## MODE SUMBER GAMBAR (WAJIB DITAWARKAN)

### MODE A (Direkomendasikan): Lokal Penuh (Offline Maksimal)
- Semua gambar di `materi/[slug]/images/`
- Di `materi.json`, gunakan path relatif:
  `<img src="images/nama-file.webp" style="width:100%;border-radius:12px;margin:16px 0;" alt="deskripsi">`

### MODE B: GitHub Hosting (Tetap Terkontrol)
- Asset tetap disimpan lokal dulu.
- Setelah push ke GitHub, gunakan:
  `https://cdn.jsdelivr.net/gh/<user>/<repo>@<branch>/materi/[slug]/images/nama-file.webp`
- Mode B butuh internet saat akses.

## ATURAN KONTEN MATERI

- Pecah materi ke 5–10 konsep (ideal). Gunakan Focus Card.
- Setiap konsep wajib punya: `id`, `title`, `icon`, `content`, `steps`, `summary`
- KaTeX: inline `$...$`, block `$$...$$`
- Bold: `**teks**` | Italic: `*teks*` | Code: `` `kode` ``

## ATURAN QUIZ

- One question per screen
- Minimal 5 soal, idealnya 10
- Setiap soal wajib punya `explanation`
- `answer` harus persis sama dengan salah satu string di `options`

## FORMAT BALASAN WAJIB

Setiap kali saya kirim materi mentah, balas dengan urutan ini:

**1. Ringkasan Modul**
Judul, slug, kategori, jumlah konsep, jumlah soal, estimasi waktu.

**LANGKAH 1**: Buat struktur folder `materi/[slug]/` beserta subfolder `images/` dan `assets/`

**LANGKAH 2**: File `index.html` — gunakan template baku di atas, ganti [JUDUL MATERI] dan [KATEGORI]

**LANGKAH 3**: File `materi.json` — konten lengkap siap paste

**LANGKAH 4**: File `quiz.json` — soal lengkap siap paste

**LANGKAH 5**: Potongan JSON untuk update `data/materi-list.json`

**LANGKAH 6**: DAFTAR ASET WAJIB DIUNDUH (format tabel):

| nama-file | lokasi-simpan | keyword internet | lisensi | dipakai di |
|---|---|---|---|---|
| diagram-contoh.webp | materi/[slug]/images/ | "..." | CC0 | konsep-2 |

**LANGKAH 7**: Instruksi validasi — cara test via local server

## TEMPLATE DAFTAR ASET (GUNAKAN FORMAT INI)

```
- nama-file: diagram-tekanan-hidrostatis.webp
- lokasi-simpan: materi/tekanan-ipa-kelas-9/images/diagram-tekanan-hidrostatis.webp
- keyword internet: "diagram tekanan hidrostatis SMP" OR "hydrostatic pressure diagram"
- lisensi disarankan: CC0 / Public Domain / Unsplash License
- dipakai di: konsep-3
```

## ATURAN KETAT OUTPUT KODE

- JSON harus valid — tidak ada trailing comma
- Path harus konsisten dengan struktur di atas
- Jangan overwrite materi lain
- Jangan mengubah struktur core app (assets/, components/, data/)
- `index.html` materi TIDAK boleh berisi hardcoded markup focus-overlay lengkap
- Wajib pakai `data-component="focus-panel"` dan `data-component="bottom-nav"`
- Init FocusCardManager WAJIB pakai pattern `app:ready` + fallback (lihat template)
